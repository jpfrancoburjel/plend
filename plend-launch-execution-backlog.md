# Plend Launch Execution Backlog

## Goal
Turn Plend from idea into a launchable local service.

## Rule
This backlog is sequenced for speed and sanity, not fantasy.

---

# Phase 1. Validate the concept

## 1. Define the first 3 packages
- Finalize Mini Deco
- Finalize Mini Play
- Finalize Signature Plend
- Decide whether Premium Plend launches immediately or slightly later

## 2. Lock the initial themes
- choose 2 to 3 reusable themes
- define what assets are reusable across themes
- avoid custom chaos

## 3. Competitor spot-check
- manually review 10 nearby competitors
- note pricing clues, visuals, weak points, positioning
- compare venue birthdays vs at-home services

## 4. Interview target customers
- speak to 5 to 10 parents in the zone
- ask what they normally spend
- ask what stresses them most
- ask what would make them book at home instead of a venue

---

# Phase 2. Build the offer

## 5. Create package sheets
- package name
- hero image reference
- inclusions
- exclusions
- starting price
- ideal customer

## 6. Define add-ons
- extra hour
- second play module
- custom signage
- larger balloon setup

## 7. Set pricing rules
- distance surcharge rules
- event duration rules
- custom work rules
- staffing thresholds by child count

---

# Phase 3. Build operations

## 8. Inventory list
- list every prop, decor piece, and play module
- mark reusable vs consumable
- estimate replacement cost

## 9. Storage system
- define bins / labels / theme grouping
- create setup checklists
- create cleaning/reset checklist

## 10. Delivery workflow
- loading checklist
- unloading checklist
- setup order
- teardown order

## 11. Safety and hygiene standard
- cleaning process after each event
- wear-and-tear inspection process
- damaged-item handling

---

# Phase 4. Build the brand and sales assets

## 12. Branding
- logo
- color palette
- typography
- visual style

## 13. Instagram setup
- create profile
- write bio
- create highlights
- prepare first 12 posts

## 14. Website or landing page
- who we are
- service zones
- package overview
- inquiry CTA
- contact / WhatsApp form

## 15. Sales deck and PDF package guide
- use the Plend deck draft
- create a client-facing version

---

# Phase 5. Source assets

## 16. Buy or source inventory
- soft play modules
- mats
- ball pit
- decor structures
- signage elements
- storage materials

## 17. Test setup in real life
- build each package once
- time the full setup and teardown
- photograph everything
- note weak points

---

# Phase 6. Launch commercially

## 18. Soft launch
- offer to first 3 to 5 bookings
- collect photos and testimonials
- monitor setup time and profitability

## 19. Referral push
- ask first clients for referrals
- partner with photographers / family venues / local communities

## 20. Content loop
- publish each event
- reuse each event into multiple posts
- gather DMs and FAQs for future content

---

# Phase 7. Optimize

## 21. Review after first 5 bookings
- which package sells most
- which package is most profitable
- what takes too long
- what customers ask for repeatedly

## 22. Fix the bottlenecks
- reduce setup complexity
- simplify weak themes
- raise prices if needed
- cut low-value custom requests

## 23. Expand only if deserved
- add one new theme
- add one stronger premium package option
- consider weekday / private venue partnerships

---

# Top priorities right now
If doing only the next few moves, do these first:
1. Finalize 3 packages
2. Lock 2 to 3 launch themes
3. Build startup budget by item
4. Create first visual references / mockups
5. Interview 5 to 10 target parents

That is the highest-leverage start.
